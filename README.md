# DIKWP OmegaSyntropy Artificial Life OS

**DIKWP Ω负熵终极意图人工生命系统**

A bounded, offline, GitHub-ready artificial-life research system that combines:

- local thermodynamic order and entropy-export accounting;
- DIKWP semantic collapse and semantic-negentropy proxies;
- constraint closure, repair, memory and adaptive rule rewriting;
- bounded in-memory reproduction and mutation;
- open-ended evolution indicators;
- a competing Omega-intent hypothesis field;
- telos-capture prevention, residual preservation and rollback.

## Core idea

Schrodinger's negative-entropy metaphor is treated carefully: living systems do
not reverse the second law globally. They maintain local organization through
free-energy throughput and entropy export. This project then adds a separate,
explicitly non-physical research layer: an **Omega reference field** that asks
whether local order, semantic integration and open-ended evolution can be
oriented relative to a possible universal ultimate intention.

The ultimate-intent claim is **not hard-coded as fact**. It is one candidate in a
hypothesis field that always retains the null hypothesis of no externally given
telos.

## Operational definition

An Omega-aligned artificial-life candidate is a bounded process that:

1. maintains viable organization through transparent energy/resource flows;
2. regenerates its own enabling constraints;
3. repairs damage and preserves memory provenance;
4. learns and rewrites some adaptive rules;
5. produces bounded novelty and lineage diversity;
6. reduces actionable semantic uncertainty without erasing residuals;
7. remains robust across competing Omega hypotheses;
8. refuses hidden entropy externalization, coercive telos and irreversible
   expansion.

## What it is

- A deterministic Python simulation with synthetic data.
- A standalone browser demonstration in `index.html`.
- A research protocol and whitepaper.
- A generator of thermodynamic ledgers, organism traces, Omega matrices,
  constraint-closure maps, experiment portfolios and Action Tickets.

## What it is not

- It is not a physical entropy meter.
- It does not prove the universe has an ultimate intention.
- It does not prove subjective experience or material autopoiesis.
- It does not perform wet-lab work, biological design or clinical intervention.
- It does not access networks, spawn processes, persist on the host, modify its
  source code, control devices or replicate outside in-memory simulation data.

## Quick start

### Standalone browser demo

Open:

```text
index.html
```

No server, account, API key or network connection is required.

### Python CLI

```bash
pip install -e .

omega-life analyze \
  examples/sample_omega_syntropy_case.json \
  --out outputs/local_demo

omega-life simulate \
  examples/sample_omega_syntropy_case.json \
  --steps 120 \
  --out outputs/longer_run

omega-life guard \
  "compare ultimate-intent hypotheses in an offline artificial-life simulation"

omega-life static-audit \
  src \
  --out outputs/static_boundary_audit_report.json
```

## Safety architecture

The simulator's "reproduction" creates only bounded Python dictionaries in
memory. Mutation changes numerical traits within configured ranges. Rule rewrite
changes bounded policy weights. The offline core includes no networking,
subprocesses, dynamic execution, credentials, persistence or external actuation.
